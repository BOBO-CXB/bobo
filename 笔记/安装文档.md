# 安装文档

### 准备工作：

- 后端文件打包、配置文件
- 前端文件编译（dist文件夹下）
- 服务器中新建一个项目文件夹  **intelligent-patrol**  在项目文件夹下新建两个子文件夹
  - **patrol-back** 用于存放后端文件
  - **patrol-front**用于存放前端文件



### 后端服务：

- 新建三个文件夹
  - bin  存放后端jar包和启动程序
  - config 存放项目配置文件
  - log 存放项目日志

- 将后端打包文件移入 **patrol-back** bin 文件夹中

- 编写启动脚本 restart.sh

- ```shell
  ps -ef |grep patrol-1.0.0-SNAPSHOT.jar |grep -v grep | awk '{printf("%s\n",$2)}'|xargs kill -15
  
  sleep 2
  
  JVM_ARGS=" -Xms512m -Xmx512m "
  
  nohup java $JVM_ARGS -jar patrol-1.0.0-SNAPSHOT.jar --spring.config.location=/home/intelligent-patrol/patrol-back/config/application.yml >>  /home/intelligent-patrol/patrol-back/log/api-server.log 2 >&1 &
  
  rm -rf nohup.out               
  ```

- 将配置文件application.yml 放入 config

- 执行 ./restart.sh 运行脚本



### 前端：

- 将前端编译好的文件放入**patrol-front**

- 调整**IPConfig.js**配置文件中的后端IP地址

- 配置nginx  配置文件路径  /usr/local/nginx/conf/conf.d

- 在conf.d下新建patrol-front.conf 项目配置文件

- ```conf
  server {
      listen       9086;
      server_name  localhost;
  
      location / {
          root   /root/intelligent-patrol/patrol-front;
          index  index.html index.htm;
      }
      location /patrolServer/ {
                  proxy_redirect off;
          proxy_pass  http://localhost:8101/;
      }
  }
  
  ```

- 修改完配置文件后重新启动nginx服务   systemctl restart nginx.service
- 查看nginx服务状态 systemctl status nginx.service  显示active (running)表示启动成功
- 通过IP+端口号即可访问前端页面
