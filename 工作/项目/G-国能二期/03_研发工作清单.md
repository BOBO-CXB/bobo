# 一期“评价分析及指标填报”优化研发工作清单

## 1. 使用说明

本清单用于研发快速定位本次改造范围和防止遗漏，不替代 PRD。详细业务规则、页面结构、字段口径和验收标准以 `01_评价分析及指标填报优化PRD.md` 为准；评价分析页面样式和交互参考 `02_评价分析HTML原型/打开评价分析原型.html`。

## 2. 本次改造范围

| 模块 | 核心改造 | 交付形式 |
|---|---|---|
| 评价分析 | 增加任务父卡片及各单位任务级完成/得分展示，简化一级指标交互，重构单位指标详情 | PRD + HTML 原型 |
| 指标填报 | 增加 0%—100% 完成进度选择，衔接原全部完成评审流程 | PRD |
| 完成率统计 | 按全部有效指标工作项完成进度等权平均 | PRD |

## 3. 研发任务索引

### 3.1 前端

- [ ] **统计条件**：评价分析页面按“统计周期、评价任务”顺序展示筛选条件，切换后即时刷新，不设置查询按钮。
- [ ] **任务父卡片**：取消顶部独立任务概览卡；左侧“评价任务与指标”区域回显当前评价任务父卡片，仅展示任务名称、完成率和得分率，并在首次进入或切换任务后默认选中。
- [ ] **一级指标**：一级指标卡片相对父任务卡片缩进并显示层级引导线；仅展示名称、完成率和得分率，取消展开箭头、下级指标和展开/收起交互。
- [ ] **单位列表**：右侧区域统一命名为“各单位完成情况”。选中父任务时展示各单位任务级状态、完成进度、得分率和更新时间，操作列显示“—”；选中一级指标时展示指标维度数据并提供查看详情。
- [ ] **列表排序**：提供“按完成率、按得分率”两种排序；默认按完成率从高到低排列，按得分率时同样从高到低排列，未评分单位置于末尾。
- [ ] **指标详情**：使用大弹框展示评价单位、一级指标汇总和人员级指标工作项；每条工作项展示名称、责任部门、责任人、完成进度、完成情况和得分。
- [ ] **指标填报**：在现有指标填报弹框增加 0%—100% 完成进度滑块，步长 1%，同步显示当前百分比；再次填报回显最近一次保存值。
- [ ] **完成处理**：0%—99% 按普通阶段填报保存；100% 等同原“全部完成”，提交后进入既有指标评审流程。
- [ ] **历史内容**：历史填报展示当次进度；导入上期填报内容不得覆盖当前已回显的完成进度。

### 3.2 后端与数据

- [ ] **进度存储**：新增或补充指标工作项完成进度字段，取值为 0—100 的整数；保存时与填报内容、附件建立同一条历史快照。
- [ ] **状态派生**：工作项是否完成由完成进度是否等于 100% 派生，不再接受独立且可能冲突的完成状态输入。
- [ ] **汇总计算**：单位、一级指标、评价任务完成率均按对应范围内全部有效工作项进度进行等权平均，不配置业务权重。
- [ ] **得分汇总**：任务整体得分率按全部单位适用评分项实得分合计除以应得总分合计计算；单位任务得分率按单个单位同一口径计算，不直接平均下级得分率。
- [ ] **任务级接口**：评价分析接口返回父任务卡片汇总数据及各单位任务级完成率、得分率、评分状态和最近更新时间。
- [ ] **刷新接口**：填报保存成功后重新计算并返回工作项、单位、一级指标和任务层级的最新完成率。
- [ ] **历史兼容**：仅有原“全部完成”标识的历史数据，已完成按 100% 转换；未完成且无进度数据按 0% 转换。
- [ ] **权限流程**：沿用现有填报人、评审和完成后不可再次填报/不可撤回规则，不新增流程状态。

### 3.3 联调与回归

- [ ] 验证首次进入及评价任务、统计周期切换后默认选中父任务卡片，父任务卡片和单位任务级列表数据一致。
- [ ] 验证父任务与一级指标切换时“当前查看”、操作列、“查看详情”入口和表格列宽符合设计。
- [ ] 验证按完成率和按得分率均从高到低排序，且按得分率排序时未评分单位位于列表末尾。
- [ ] 验证工作项填报 0%、中间进度、99% 和 100% 四类场景。
- [ ] 验证 100% 提交能够进入原评审流程，0%—99% 不触发完成评审。
- [ ] 验证多条工作项不同进度下，单位、一级指标和任务完成率的等权平均结果正确。
- [ ] 验证历史填报查看、上期内容导入、附件上传及历史数据兼容转换。
- [ ] 验证一期现有导航、评价任务选择、评分和审批功能未受到本次改造影响。

## 4. 不在本次范围

- 不调整评价指标配置、评分规则和既有评审状态定义。
- 不新增部门下钻、催办、导出或批量处理。
- 不制作指标填报弹框 HTML 原型，以 PRD 页面说明和线框图为实现依据。
