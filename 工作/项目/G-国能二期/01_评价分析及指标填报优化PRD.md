## 2. 需求背景

现有评价分析页面以一级评价指标为主要分析入口，左侧展示一级指标及其完成率、得分率，右侧展示各单位在当前指标下的完成情况。当前设计存在以下问题：

1. 页面缺少评价任务层级的整体完成情况，用户只能逐项查看一级指标，无法快速判断本次评价任务总体进度。
2. 左侧一级指标卡片继续展开下级指标，与右侧“查看详情”入口形成重复的信息下钻路径，增加页面复杂度和理解成本。
3. 指标填报弹框仅提供“指标项填报全部完成”复选框，无法记录指标工作项在 1%—99% 区间内的阶段进展，与评价分析页面按工作项完成进度汇总的统计口径不匹配。
4. 现有任务概览仅反映评价任务总体完成率，右侧单位列表始终按一级指标展示，缺少各单位在整个评价任务下的完成率、得分率和更新时间。

## 3. 需求目标

- 补充评价任务最高层级的整体完成率，使管理人员进入页面后能够优先判断任务总体进展。
- 建立“评价任务—一级指标”的父子分析层级，支持在同一单位列表中切换查看任务维度和指标维度数据。
- 明确左侧指标区和右侧详情区的信息分工，减少重复展开操作。
- 支持责任人在每次指标填报时更新完成进度，使填报数据能够连续反映实际推进情况，并作为上层完成率的统计依据。
- 保持一期系统顶部导航、左侧菜单及评价分析主从布局不变，控制本轮改造范围。

## 4. 方案评估与结论

### 4.1 评价任务整体完成率

#### 4.1.1 方案评估

| 方案 | 说明 | 评估 |
|---|---|---|
| 一级指标完成率算术平均 | 将页面展示的一级指标完成率直接平均 | 未考虑不同一级指标包含的工作项数量差异，不能准确反映全部工作项的实际进度 |
| 按工作项完成状态统计 | 仅统计已达到 100% 的工作项数量占比 | 1%—99% 的进行中工作项无法反映到上层完成率，进度表达不连续 |
| 全部有效工作项完成进度等权平均 | 汇总全部有效指标工作项的上报完成进度，每条工作项等权参与计算 | 无须配置指标权重，能够连续反映各工作项的实际推进情况，符合当前业务管理要求 |

#### 4.1.2 结论

采用“全部有效指标工作项完成进度等权平均”口径，由服务端计算并返回评价任务整体完成率。业务不配置指标权重，每条有效工作项的计算权重均为 1。

> 评价任务整体完成率 = 全部有效指标工作项完成进度之和 ÷ 有效指标工作项数量

- **指标工作项**：评价任务落实到具体单位、责任部门和责任人的最末级工作记录。
- **有效工作项**：已纳入当前评价任务且不属于“不适用”、已取消或无须填报的工作项。
- **完成进度**：由责任人填报，取值范围为 0%—100%；责任人即该工作项的填报人。
- **汇总规则**：一级指标完成率、单位完成率和任务整体完成率均以各自统计范围内的有效工作项完成进度进行等权平均，不设置业务权重。
- **精度规则**：计算值保留一位小数；页面可按展示空间显示整数或一位小数，原型示例显示整数。

前端直接使用任务接口返回的 `taskCompletionRate` 汇总字段，不自行对页面可见的一级指标完成率进行二次平均。

### 4.2 左侧评价指标交互简化

左侧卡片展开下级指标与右侧单位详情均承担信息下钻功能，存在路径重复。左侧展开后还会增加列表高度、嵌套边框和视觉密度，降低一级指标之间的对比效率。

结论：取消左侧一级指标卡片的展开/收起交互。左侧仅展示一级指标名称、完成率和得分率，并承担一级指标切换功能；最末级指标及单位评价明细统一由右侧单位列表的“查看详情”承载。

### 4.3 指标填报完成进度

原“指标项填报全部完成”复选框只能表达未完成、已完成两种结果，不能支持责任人按月或按阶段持续更新进度。若在复选框之外再增加一套独立完成进度，会产生“已勾选全部完成但进度不足 100%”等冲突状态。

结论：以 0%—100% 的完成进度作为指标工作项的唯一进度值。原“指标项填报全部完成”不再作为独立数据字段；当完成进度提交为 100% 时，等同原勾选“全部完成”，继续触发既有指标评审流程和完成后的限制规则。

### 4.4 任务父卡片与单位任务汇总

顶部独立任务概览卡与评价任务筛选、左侧分析卡片存在信息重复，且无法自然承接各单位在整个任务下的汇总结果。为保持现有主从结构，选中的评价任务直接回显在左侧卡片区，作为一级指标的父级分析卡片。

- **层级结构**：左侧区域统一命名为“评价任务与指标”。父任务卡片全宽展示，一级指标卡片整体缩进，并使用低对比度竖向引导线表达父子关系。
- **展示一致性**：父任务卡片与一级指标卡片均只展示名称、完成率和得分率，不增加状态、辅助统计、展开箭头或额外业务标签。
- **默认维度**：页面首次进入以及切换评价任务后，默认选中父任务卡片，右侧展示各单位在整个评价任务下的汇总结果。
- **操作边界**：任务维度不提供详情下钻，操作列显示“—”；一级指标维度继续提供“查看详情”，进入人员级指标工作项明细。
- **信息去重**：取消筛选栏下方独立任务概览卡，顶部仅保留统计周期和评价任务筛选条件。

## 5. 功能需求

### 5.1 统计条件与评价任务父卡片

页面顶部仅保留常规筛选栏，先展示统计周期，再展示评价任务。选中的评价任务在左侧“评价任务与指标”区域回显为父任务卡片，任务完成率和得分率统一在该卡片中展示。

- **统计周期**：由年份和月份两个单选控件组成，位于筛选栏首位；“分析周期”统一更名为“统计周期”。
- **评价任务**：使用常规下拉控件，位于统计周期之后；可选任务范围根据所选统计周期加载。
- **当前任务回显**：父任务卡片展示完整任务名称、任务完成率和任务得分率，不重复展示任务状态及辅助统计。
- **整体完成率**：展示当前评价任务、当前统计周期下全部有效工作项完成进度的等权平均值。
- **整体得分率**：按当前任务全部单位、全部适用评分项的实得分合计除以应得总分合计计算，不直接平均各单位得分率。
- **即时刷新**：切换评价任务、年份或月份后，整体完成率与下方指标、单位列表同步刷新，不设置独立查询按钮。
- **默认选中**：页面首次进入或切换评价任务后，自动选中父任务卡片，并将右侧列表切换为任务维度。
- **进度表达**：父任务卡片和单位表格均使用百分比，不增加进度条，保持与一级指标卡片一致。

| 条件 | 状态 |
|---|---|
| 完成率 = 0% | 未开始 |
| 0% < 完成率 < 100% | 进行中 |
| 完成率 = 100% | 已完成 |

### 5.2 评价任务与一级指标列表

- **展示内容**：父任务卡片及其下属一级指标卡片均展示名称、完成率、得分率。
- **父子表达**：父任务卡片保持全宽；一级指标卡片向右缩进 12—16px，并通过竖向引导线与父任务形成层级关系。
- **默认选中**：页面首次进入时默认选中父任务卡片。
- **任务切换**：点击父任务卡片后，右侧单位列表展示各单位在整个任务下的完成率、得分率和更新时间。
- **切换交互**：点击任一一级指标卡片后更新选中状态，右侧单位列表同步展示该指标下的单位数据。
- **交互限制**：不显示展开箭头，不在卡片内部展示下级指标，不支持展开或收起。
- **选中反馈**：选中指标使用左侧强调色条和背景差异表达。

### 5.3 各单位完成情况

- **当前查看**：任务维度显示当前评价任务名称；指标维度显示当前选中的一级指标名称。
- **列表字段**：序号、单位名称、状态、完成进度、得分率、更新时间、操作。
- **任务维度**：展示各单位在整个任务下的完成进度、得分率和最近更新时间。单位任务完成率按该单位在任务下全部有效工作项进度等权平均；单位任务得分率按该单位全部适用评分项的实得分合计除以应得总分合计计算。
- **得分状态**：无评分记录显示“暂未评分”；部分评分时展示当前得分率并标记“评分中”；全部评分完成后展示最终得分率。
- **操作限制**：任务维度保留操作列但显示“—”，不展示可点击或置灰的“查看详情”，避免误解为权限不足，并保持任务/指标切换时表格列宽稳定。
- **指标详情**：仅在一级指标维度提供“查看详情”，点击后打开“指标详情”大弹框，展示当前单位在所选一级指标下落实到责任人的指标工作项明细。
- **排序方式**：提供“按完成率”和“按得分率”两种排序；默认按完成率从高到低排列，切换为按得分率时同样从高到低排列，未评分单位统一置于列表末尾。

### 5.4 指标详情

指标详情采用大尺寸弹框，承接当前单位、当前一级指标下的人员级工作项信息，不再显式展示次级指标。

- **弹框规格**：宽度约为浏览器视口的 90%，最大宽度约 1320px；最大高度不超过视口的 85%。标题栏与上部汇总区固定，下部工作项区域独立滚动。
- **弹框标题**：统一显示“指标详情”，标题栏不重复展示评价单位名称；单位名称仅在上部汇总区展示一次。
- **上部汇总**：展示评价单位、一级指标名称、完成率和得分率。
- **工作项名称**：展示落实到责任人的完整指标工作项名称。
- **责任信息**：展示责任部门和责任人；责任人同时作为该工作项的填报人，不重复设置“填报人”字段。
- **完成进度**：展示责任人填报的 0%—100% 完成进度。
- **完成情况**：分为“总体开展情况”和“本月开展情况”两部分展示，支持长文本自动换行。
- **得分**：按“实得分 / 该条总分”展示，例如“0.8 / 1.0 分”；尚未评分时显示“待评分”，不以 0 分替代。
- **更新时间**：属于辅助追溯信息，页面空间允许时可展示；空间不足时可不展示，不影响本轮核心功能验收。
- **层级保留**：界面不展示次级指标，但数据中仍保留工作项所属指标层级，用于配置、统计和追溯。
- **关闭方式**：支持关闭按钮及点击遮罩关闭。

### 5.5 指标填报进度更新

指标填报弹框在现有填报内容、历史内容查看和附件上传能力基础上，增加完成进度选择，使责任人可在每次填报时同步更新当前指标工作项的推进比例。

- **字段位置**：在弹框顶部说明区与填报内容区之间增加“完成进度”字段，作为本次填报的必填项。
- **控件形式**：采用 0%—100% 的横向进度滑块，步长为 1%，滑块右侧同步展示当前百分比数值。
- **默认回显**：首次填报默认显示 0%；再次填报时回显该工作项最近一次已保存的完成进度，责任人可在此基础上继续调整。
- **保存规则**：点击“确定”时，将完成进度与本次填报内容、附件一并保存；每次提交均形成历史填报记录，历史记录保留当次完成进度快照。
- **进行中填报**：完成进度为 0%—99% 时，按普通阶段填报保存，不触发指标完成评审，后续统计周期仍可继续填报和更新进度。
- **全部完成**：完成进度为 100% 时，等同原“指标项填报全部完成”。提交前展示原有完成提示，确认后进入既有指标评审流程；评审通过后的不可再次填报、不可撤回等规则保持不变。
- **上期内容**：“导入上期填报内容”沿用原功能既有导入范围，但不得覆盖当前工作项已回显的完成进度；“查看上期填报内容”在原有内容基础上补充展示上期完成进度。
- **统计刷新**：保存成功后，当前工作项完成进度立即更新，并由服务端重新计算对应单位、一级指标及评价任务的完成率。
- **兼容处理**：历史数据若仅存在“全部完成”标识，则已完成记录转换为 100%，未完成且无进度记录的数据按 0% 处理。
- **原功能保持**：总体开展情况、本月开展情况、附件上传、取消操作和既有审批权限不因本次改造发生变化。

## 7. 数据要求

| 字段 | 说明 |
|---|---|
| taskId | 评价任务标识 |
| taskCompletionRate | 评价任务整体完成率 |
| taskScoreRate | 评价任务整体得分率；全部单位适用评分项实得分合计 / 应得总分合计 |
| indicatorId | 一级指标标识 |
| indicatorCompletionRate | 一级指标完成率 |
| indicatorScoreRate | 一级指标得分率 |
| unitTaskCompletionRate | 单位在当前评价任务下的完成率 |
| unitTaskScoreRate | 单位在当前评价任务下的得分率 |
| scoreStatus | 评分状态：暂未评分、评分中、已评分 |
| unitTaskUpdatedAt | 单位任务汇总数据最近更新时间 |
| detailIndicatorList | 当前单位在所选一级指标下的明细指标列表 |
| completionProgress | 指标工作项当前完成进度，整数，取值范围 0—100 |
| completionStatus | 是否已完成；由 `completionProgress = 100` 派生，不独立填报 |
| progressUpdatedAt | 当前完成进度最近保存时间 |
| reportHistory.progress | 历史填报记录对应的完成进度快照 |

## 8. 验收标准

1. 页面顶部按“统计周期、评价任务”的顺序展示常规筛选条件，并将“分析周期”统一更名为“统计周期”。
2. 顶部不再展示独立任务概览卡；左侧“评价任务与指标”区域回显父任务卡片，展示任务名称、完成率和得分率。
3. 页面首次进入及切换评价任务后默认选中父任务卡片，右侧展示各单位任务级完成率、得分率和更新时间。
4. 一级指标卡片相对父任务卡片缩进并显示层级引导线，不显示展开箭头和下级指标。
5. 右侧区域标题显示“各单位完成情况”，“当前查看”与左侧选中的任务或一级指标保持一致。
6. 任务维度操作列显示“—”；一级指标维度“查看详情”能够打开指标详情大弹框，并展示对应的多条人员级指标工作项。
7. 每条工作项至少展示指标项名称、责任部门、责任人、完成进度、完成情况和“实得分 / 总分”。
8. 任务、单位和一级指标完成率均按照统计范围内全部有效工作项完成进度进行等权平均，不配置业务权重。
9. 父任务卡片、一级指标卡片和单位列表均以百分比表达完成进度，不在单位表格中恢复进度条。
10. 单位列表默认按完成率从高到低排列；切换为按得分率后按得分率从高到低排列，未评分单位置于末尾。
11. 页面保持一期系统整体结构，不新增单位搜索或独立查询操作。
12. 指标填报弹框展示 0%—100% 完成进度滑块和当前百分比，完成进度为必填项，再次填报时回显最近一次保存值。
13. 完成进度为 0%—99% 时可正常保存并继续后续填报，不触发指标完成评审；完成进度为 100% 时进入原“全部完成”对应的评审流程。
14. 保存进度后，工作项及对应单位、一级指标、评价任务的完成率按照等权平均口径同步更新。
15. 历史填报记录能够查看当次完成进度；导入上期内容不得覆盖当前完成进度。
16. 原填报内容、附件上传和审批权限保持可用，历史二值完成数据能够按 0% 或 100% 兼容转换。

## 9. 本轮边界

- 本轮不调整评价指标配置、评分规则和评价流程状态定义。
- 本轮不新增部门下钻、催办、导出或批量处理功能。
- 本轮不新增任务维度单位详情弹框；任务的进一步拆解通过左侧一级指标卡片完成。
- 本轮 HTML 原型仅覆盖评价分析页面；指标填报弹框改造以本 PRD 为研发依据，不另行制作 HTML 原型。
- 评价分析原型使用模拟数据表达交互，正式完成率、工作项进度及明细数据以接口返回为准。
